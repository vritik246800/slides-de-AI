[01] OBJETIVO DO TRABALHO
O que foi implementado
[BFS]BUSCA EM LARGURA

Busca não informada. FIFO. Menor nº de passos. Ignora custos de terreno.

[GRD]GREEDY BEST-FIRST

Busca informada. Usa h(n). Rápido mas não garante menor custo total.

[A*]A-ESTRELA

Busca informada. f(n)=g(n)+h(n). Ótimo e completo com h admissível.

O programa permite desenhar o mapa (obstáculos, terreno difícil, armadilhas, inimigos), definir início/objetivo, executar os algoritmos e observar o caminho e as métricas.
[02] FUNDAMENTOS TEÓRICOS
Problema de Busca

Forma formal de descrever navegação num espaço de possibilidades com 4 elementos:

ComponenteSignificado
Estado inicialPosição de início do agente
ObjetivoPosição que o agente quer alcançar
AçõesMovimentos ortogonais ↑ ↓ ← →
Custo de passoCusto de entrar na célula vizinha
O algoritmo explora a partir do início, expandindo vizinhos, até encontrar o objetivo.

[03] TIPOS DE BUSCA
Não Informada vs. Informada
[NÃO INFORMADA]SEM HEURÍSTICA

Explora sem saber onde está o objetivo. Sem informação sobre distância restante.

BFS — FIFO
DFS — LIFO

Ordenação: FIFO (BFS) · LIFO (DFS)

[INFORMADA]COM HEURÍSTICA h(n)

Usa h(n) que estima custo de n ao objetivo para guiar a busca eficientemente.

Greedy — f(n)=h(n)
A* — f(n)=g(n)+h(n)

Heurística: Distância de Manhattan

h(n) = |linha_n − linha_obj| + |coluna_n − coluna_obj|
[04] ALGORITMO — BUSCA NÃO INFORMADA
BFS — Busca em Largura

Explora todos os nós a uma profundidade antes de avançar. Estrutura: collections.deque (FIFO).

  1. Coloca estado inicial numa fila FIFO
  2. Retira o primeiro estado e expande vizinhos
  3. Vizinhos não visitados adicionados ao fim
  4. Repete até objetivo ou fila vazia
f(n) = profundidade(n) · FIFO puro
[PROPRIEDADES]BFS
CompletoSim
Ótimo passosSim
Ótimo custoNão — ignora terreno
MemóriaAlta — O(b^d)
HeurísticaNenhuma
Ficheiroalgorithms/bfs.py
BFS garante o caminho mais curto em passos, mas não em custo quando há terrenos com custos diferentes.
[05] ALGORITMO — BUSCA INFORMADA
Greedy Best-First

Sempre expande o nó mais próximo do objetivo segundo h(n). Estrutura: heapq (min-heap).

  1. Coloca estado inicial na heap com prioridade h(n)
  2. Retira sempre o nó com menor h(n)
  3. Expande vizinhos e insere na heap com h(n)
  4. Repete até objetivo ou heap vazia
f(n) = h(n)
[PROPRIEDADES]GREEDY
VelocidadeAlta — poucas expansões
Ótimo custoNão
CompletoNão sempre
LabirintosPode ficar preso
HeurísticaManhattan h(n)
Ficheiroalgorithms/greedy.py
Trade-off: Muito rápido mas pode encontrar caminhos caros — não considera g(n).
[06] ALGORITMO — BUSCA INFORMADA ÓTIMA
A* — A-Estrela

Combina o custo real g(n) com a estimativa h(n). Algoritmo de referência — ótimo e completo.

  1. Coloca estado inicial na heap com f(n)=g(n)+h(n)
  2. Retira sempre o nó com menor f(n)
  3. g_novo = g(atual) + custo_passo para cada vizinho
  4. Se g_novo melhora custo conhecido, atualiza
f(n) = g(n) + h(n)
[SÍMBOLOS]A*
g(n)Custo real desde início até n
h(n)Estimativa heurística de n ao obj.
f(n)Custo total estimado
[PROPRIEDADES]A*
CompletoSim
Ótimo custoSim (h admissível)
EficiênciaMelhor que BFS
Ficheiroalgorithms/astar.py
